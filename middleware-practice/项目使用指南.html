中间件实操项目 · 使用指南

Kafka + Elasticsearch + HBase + Qdrant — 基于 Docker 的本地实操环境

一、项目结构

middleware-practice/
├── pom.xml                     # 父 POM (Spring Boot 3.4.5 + JDK 21)
├── docker-compose.yml          # 一键启动所有中间件
├── common/                     # 公共模块 (统一响应 ApiResponse)
├── kafka-demo/     :8081       # Kafka 消息队列实操
├── es-demo/        :8082       # Elasticsearch 搜索实操
├── hbase-demo/     :8083       # HBase 分布式存储实操
├── qdrant-demo/    :8084       # Qdrant 向量检索实操
└── xxl-job-demo/   :8085       # XXL-JOB 执行器接入实操

二、启动方式

前提条件

安装 Docker Desktop 并启动（菜单栏出现 Docker 图标即表示运行中）
安装 JDK 21 和 Maven 3.6+

启动中间件（Docker）

# 1. 进入项目目录
cd middleware-practice

# 2. 启动所有中间件（Kafka + ES + Kibana + HBase + Qdrant）
docker-compose up -d

# 3. 检查是否全部启动成功
docker-compose ps

# 期望看到 6 个容器都是 Up 状态：
#   zookeeper, kafka, elasticsearch, kibana, hbase, qdrant

启动 Java 应用

# 在 middleware-practice 目录下：

# 编译整个项目
mvn clean compile

# 统一推荐：从根目录启动模块（会自动带上依赖模块 common）
mvn -pl kafka-demo  -am spring-boot:run -DskipTests    # → http://localhost:8081
mvn -pl es-demo     -am spring-boot:run -DskipTests    # → http://localhost:8082
mvn -pl hbase-demo  -am spring-boot:run -DskipTests    # → http://localhost:8083
mvn -pl qdrant-demo -am spring-boot:run -DskipTests    # → http://localhost:8084

# 不建议进入子模块目录后直接 mvn spring-boot:run
# 原因：kafka-demo/es-demo/hbase-demo/qdrant-demo 都依赖 common，
# 单独跑子模块时，Maven 可能把 common 当外部依赖解析，导致启动失败。

IntelliJ IDEA 启动方式

# 正确姿势
# 1. 用项目根目录的 pom.xml 导入整个多模块工程
#    middleware-practice/pom.xml
# 2. 确认 IDEA 里能看到 common、kafka-demo、es-demo、hbase-demo、qdrant-demo 五个模块
# 3. 直接点 Application 主类左边绿色三角启动，不要点 Maven 的 spring-boot:run

# 对应主类
com.example.kafka.KafkaDemoApplication
com.example.es.EsDemoApplication
com.example.hbase.HBaseDemoApplication
com.example.qdrant.QdrantDemoApplication
com.example.xxljob.XxlJobDemoApplication

# 如果 IDEA 点击绿色三角仍然失败，通常是因为工程不是按“整个多模块项目”导入，
# 而是只导入了某个子模块，导致 common 没进入运行时 classpath。

三、API 接口速查

XXL-JOB 模块 (:8085) —— 分布式任务调度执行器

模拟场景：同一份“日报生成”逻辑，分别支持本地 @Scheduled 和平台化 @XxlJob 触发，用于理解两者差异和接入方式。

GET /api/jobs/status
GET /api/jobs/report-preview
# 启动 XXL-JOB Demo（默认只演示 Executor）
mvn -pl xxl-job-demo -am spring-boot:run -DskipTests

# 打开本地 @Scheduled 对照任务
mvn -pl xxl-job-demo -am spring-boot:run -DskipTests \
  -Dspring-boot.run.profiles=local-scheduled

# 查看 demo 状态
curl http://localhost:8085/api/jobs/status

# 预览报表任务执行结果
curl http://localhost:8085/api/jobs/report-preview

# 默认配置会连接本地 XXL-JOB Admin
# http://localhost:8080/xxl-job-admin

Kafka 模块 (:8081) —— 消息队列

模拟场景：用户下单 → 发 Kafka 消息 → 短信服务/积分服务/分析服务并行消费

POST /api/orders
# 创建订单 + 发消息
curl -X POST http://localhost:8081/api/orders \
  -H 'Content-Type: application/json' \
  -d '{"userId":"user_123","product":"iPhone15","amount":8999}'

# 看控制台输出：三个 Consumer Group 分别收到消息

Elasticsearch 模块 (:8082) —— 全文搜索

模拟场景：商品索引 + 分词搜索 + 范围过滤 + 排序

POST /api/products
GET /api/products/search?keyword=...&category=...&minPrice=...&maxPrice=...
GET /api/products/{id}
DELETE /api/products/{id}
# 索引商品
curl -X POST http://localhost:8082/api/products \
  -H 'Content-Type: application/json' \
  -d '{"title":"苹果iPhone 15 Pro 256G 原色钛金属","category":"手机","price":8999,"stock":120}'

curl -X POST http://localhost:8082/api/products \
  -H 'Content-Type: application/json' \
  -d '{"title":"苹果 MacBook Pro 14英寸 M3芯片","category":"笔记本","price":14999,"stock":80}'

# 分词搜索（中文参数建议用 --data-urlencode）
curl -G http://localhost:8082/api/products/search \
  --data-urlencode 'keyword=苹果手机' \
  --data-urlencode 'minPrice=5000' \
  --data-urlencode 'maxPrice=10000'

# 按分类过滤
curl -G http://localhost:8082/api/products/search \
  --data-urlencode 'category=笔记本'

HBase 模块 (:8083) —— 海量对话存储

模拟场景：智能客服对话记录存储，RowKey = MD5(user_id)前4位 + user_id + 反向时间戳

POST /api/chat
GET /api/chat/history?userId=...&limit=...
GET /api/chat/{rowKey}
# 保存对话
curl -X POST http://localhost:8083/api/chat \
  -H 'Content-Type: application/json' \
  -d '{"userId":"user_123","question":"我的快递怎么还没到？","answer":"您的快递预计明天到达，运单号SF123456"}'

# 查询用户历史对话（最新在前的范围扫描）
curl "http://localhost:8083/api/chat/history?userId=user_123&limit=20"

# 按 RowKey 精确查询
curl "http://localhost:8083/api/chat/{返回的rowKey}"

# 注意：HBase 容器首次启动需要约 30 秒初始化
# 如果报连接错误，稍等一会再试

Qdrant 模块 (:8084) —— 向量语义搜索

模拟场景：FAQ 语义搜索 — "快递没到"能匹配到"物流超时如何查询"

POST /api/faq
GET /api/faq/search?q=...&limit=...
GET /api/faq/search-filtered?q=...&category=...
# 插入 FAQ（向量由 mock 生成，实际替换为 BGE/m3e 模型）
curl -X POST http://localhost:8084/api/faq \
  -H 'Content-Type: application/json' \
  -d '{"question":"物流超时如何查询？","answer":"打开APP→我的→帮助中心→物流查询","category":"物流"}'

curl -X POST http://localhost:8084/api/faq \
  -H 'Content-Type: application/json' \
  -d '{"question":"如何申请退款？","answer":"进入订单详情→点击申请退款→选择原因→提交","category":"售后"}'

# 语义搜索（中文参数建议用 --data-urlencode）
curl -G http://localhost:8084/api/faq/search \
  --data-urlencode 'q=我的快递还没到怎么办' \
  --data-urlencode 'limit=3'

# 语义搜索 + 分类过滤
curl -G http://localhost:8084/api/faq/search-filtered \
  --data-urlencode 'q=怎么退钱' \
  --data-urlencode 'category=售后' \
  --data-urlencode 'limit=3'

# 如果命令行里直接输入中文 JSON 容易被换行打断，推荐先写到文件再提交：
cat > /tmp/faq.json <<'EOF'
{"question":"物流超时如何查询？","answer":"打开APP→我的→帮助中心→物流查询","category":"物流"}
EOF
curl -X POST http://localhost:8084/api/faq \
  -H 'Content-Type: application/json' \
  --data-binary @/tmp/faq.json
关于 Qdrant 的 mock Embedding：演示代码中用 Random 生成伪向量，所以搜索结果是随机的，仅用于验证 API 通路。
实际接入时，替换 mockEmbedding() 方法为 BGE/m3e 模型的 HTTP 调用即可获得正确的语义搜索结果。

四、常用排查命令

# 查看所有容器状态
docker-compose ps

# 查看某个容器的日志
docker-compose logs -f kafka
docker-compose logs -f elasticsearch

# 进入 Kafka 容器测试
docker exec -it middleware-practice-kafka-1 bash
kafka-console-consumer --bootstrap-server localhost:9092 --topic order-events --from-beginning

# 测试 ES 是否就绪
curl http://localhost:9200

# 查看 HBase Master UI（容器启动后需等约 30s）
open http://localhost:16010

# 停止所有容器
docker-compose down
Qdrant 特殊说明：如果你的 Mac 是 Apple Silicon (M1/M2/M3)，首次启动 Qdrant 需要在 Docker Desktop 中启用 Use Rosetta for x86/amd64 emulation，或使用 qdrant/qdrant:v1.9.4 镜像（已支持 ARM64）。
